-------- Feedback ---------

What you got right:
- [good]  Task is stated clearly — Claude knows what to build.
- [good]  "Plan before writing" instruction — this is agentic thinking. Most beginners skip this entirely.
- [good]  "Verify the result" — you instinctively told Claude to check its own work. That's above average.
- [bonus] "Only when fully confident" — you added a quality gate. Solid instinct.

What's missing:
- [missing] No edge case handling. What if the list is empty? What if someone types letters instead of numbers? Claude will guess — and might guess wrong.
- [missing] No output format specified. Should it print the result? Return it from a function? Save to a file? Claude will decide for you.
- [missing] "Take from user" is vague. Does the user type numbers one by one? Comma-separated? Space-separated? Claude will assume.
- [missing] No constraint on built-ins. Should Claude use max() or write the logic manually? For learning purposes this matters a lot.

- Day 1 lesson: A good Claude Code prompt answers 5 questions — What to build, How input arrives, What output looks like, What can go wrong, and What constraints apply. Your prompt answered 1.5 of 5. Your agentic instruction (plan → implement → verify) was genuinely strong — that thinking pattern will serve you well as complexity grows.
